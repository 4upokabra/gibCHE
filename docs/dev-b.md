# ТЗ Dev B — Attack Engine

## Цель
Разработать модуль активных атак: автоматизация брутфорса, SQL-инъекций, запуска эксплойтов (Metasploit) и поддержка режимов Black/Grey/White box.

## Основные задачи
1. Подключить Metasploit RPC (или аналог) и реализовать слой абстракции для запуска шаблонов атак с параметрами из backend.
2. Создать модуль брутфорса (Hydra/pwntools/кастом) для паролей, доменов и сервисов; предусмотреть управление словарями и ограничениями нагрузки.
3. Интегрировать sqlmap API (или собственный исполнитель) для SQLi: выбор payload, обработка результатов, экспорт отчётов.
4. Реализовать проверку устаревших версий ПО: сопоставление баннеров/версий с CVE/БДУ и автоматический запуск релевантных эксплойтов.
5. Добавить поддержку трёх моделей тестирования: управление доступом к исходникам/учёткам, профили конфигураций, контроль ограничений.
6. Логирование каждой операции атаки (запрос, ответ, статус), сбор артефактов и безопасное хранение.

## API и интеграции
- gRPC/REST эндпоинт `attack/run` (принимает тип атаки, набор целей, профиль модели тестирования, SLA).
- Очередь сообщений от backend (Dev D) с заданиями; события прогресса через WebSocket/SignalR.
- Обогащение данных CVE/БДУ через общий сервис или локальный кэш (координация с Dev C).

## Технические требования
- Изоляция атакующих инструментов через контейнеры/неймспейсы; настройка лимитов CPU/RAM.
- Возможность dry-run (валидировать параметры без фактического запуска).
- Планировщик повторных попыток с экспоненциальной задержкой и ручным флагом отмены.

## Критерии готовности
- Минимум три типа атак (брутфорс, SQLi, Metasploit-эксплойт) воспроизводимы через API.
- Логи и артефакты доступны backend-у и аналитике.
- Документация по профилям тестирования и примеры сценариев для презентации.
