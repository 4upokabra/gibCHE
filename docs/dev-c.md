# ТЗ Dev C — LLM & аналитика

## Цель
Создать LLM-сканер и аналитический слой: парсинг веб-страниц, генерация отчётов, категоризация уязвимостей по БДУ ФСТЭК и выдача подсказок/рекомендаций.

## Основные задачи
1. Разработать пайплайн LLM-сканера: загрузка страницы (Playwright/Requests+BS4) → очистка/структурирование → промпт → пост-обработка.
2. Настроить prompt-цепочки (разведка, оценка риска, рекомендации) для DeepSeek через OpenRouter, учесть токен-лимиты и стоимость.
3. Реализовать модуль категоризации уязвимостей: сопоставление результатов сканов/атак с БДУ ФСТЭК и CVE, приоритезация.
4. Генерировать консолидированные отчёты (Markdown/PDF/JSON) для UI и презентации; поддержать экспорт по шаблонам.
5. Встроить подсказки и Q&A интерфейс для фронтенда/бота: контекстные ответы, навигация по результатам.
6. Подготовить датасет для демо (анонимизированные примеры целей) и скрипты репликации.

## API и интерфейсы
- Сервис `analysis-LLM` с эндпоинтами: `scan/page`, `analyze/findings`, `report/generate`, `assist/query`.
- Поддержка вызовов из backend (Dev D) и прямой интеграции с UI (Dev N) через WebSocket/Streaming для длинных ответов.
- Контракты на вход: нормализованные данные от Dev A и события атак от Dev B.

## Технические требования
- Базовый провайдер — DeepSeek через OpenRouter; предусмотреть переключение на альтернативы (OpenAI, Azure, локальная Llama) через конфигурацию.
- Кеширование ответов, защита от prompt injection, ограничение на PII.
- Писать unit-тесты для парсеров и пост-обработки (pytest/uvu).

## Критерии готовности
- LLM-сканер формирует отчёт минимум по трём тестовым страницам и выдаёт категории БДУ.
- API аналитики документирован (Swagger/AsyncAPI) и покрыт примерами запросов/ответов.
- Подсказки доступны в UI/боте, есть шаблоны презентационных summary.
