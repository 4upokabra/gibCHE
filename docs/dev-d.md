# ТЗ Dev D — Backend приложений

## Цель
Реализовать серверную часть продукта: единый API для фронтенда и внутренних модулей, оркестрацию задач сканирования/атак, управление пользователями и профилями тестирования.

## Основные задачи
1. Выбрать и поднять основной стек (FastAPI/NestJS/Go), настроить структуру проекта, middlewares, систему конфигураций.
2. Реализовать REST/gRPC API:
   - аутентификация и авторизация (JWT/OAuth, роли: admin, оператор, read-only);
   - CRUD по целям, заданиям сканирования/атак, шаблонам Black/Grey/White box;
   - эндпоинты статусов и агрегации результатов для UI.
3. Организовать оркестратор задач: постановка в очередь (Redis/Rabbit), отслеживание прогресса, ретраи, отмена.
4. Интегрировать сервисы Dev A/B/C:
   - проксирование команд сканера и атак через API/очереди;
   - приём результатов, валидация, сохранение в БД;
   - прокидывание данных в LLM-аналитику и обратно в UI.
5. Добавить WebSocket/SSE слой для real-time обновлений (статус сканов, прогресс атак, подсказки LLM).
6. Подготовить систему журналирования (audit log), трейсинга и экспорт метрик (OpenTelemetry/Prometheus).

## Требования к данным и безопасности
- RBAC с гранулярными разрешениями на запуск сканов/атак.
- Шифрование чувствительных секретов (API ключи, креды) и безопасное хранение (Vault/KeyVault/Local env).
- Ограничение скорости и лимиты запросов для внешних клиентов/бота.

## интеграции
- UI/бот (Dev N) через REST/WebSocket.
- Core-сканеры (Dev A) и Attack Engine (Dev B) через очереди и сервисные учетные записи.
- LLM аналитика (Dev C) через gRPC/REST вызовы с streaming ответами.

## Критерии готовности
- Все ключевые флоу (создать цель → запустить скан → выполнить атаки → получить отчёт) проходят через один API.
- Настроена документация (Swagger/OpenAPI + коллекция в Postman) и скрипты миграций БД.
- CI-пайплайн с линтингом, тестами и сборкой контейнера backend.
