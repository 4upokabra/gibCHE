# ReconScope MVP

Командный проект для хакатона: веб-платформа активной и пассивной разведки, автоматизации атак и аналитики уязвимостей с поддержкой LLM-подсказок и визуализацией результатов.

## Описание проекта

- **Сканирование**: агрегируем результаты базовых сканеров (`nmap`, `Shodan`, `VirusTotal`, собственные пассивные/активные модули), дополнительно запускаем LLM-сканер (парсинг страниц → обработка LLM → отчёт).
- **Атаки**: поддержка брутфорса паролей/доменов, автоматизации SQL-инъекций и запуска готовых эксплойтов (в т. ч. через Metasploit-пайплайн) с выбором модели тестирования (Black/Grey/White box).
- **Оценка**: пост-обработка результатов LLM-ом, категоризация уязвимостей по БДУ ФСТЭК, приоритезация и рекомендации.
- **Приложение**: единый веб-интерфейс/Telegram-бот, визуализации графов инфраструктуры, тепловые карты рисков, дешборды прогресса атак.

## План разработки MVP

1. **Discovery & архитектура (день 1)**
   - Уточнить сценарии Black/Grey/White Box, требования к безопасности, список внешних API.
   - Спроектировать модульную архитектуру: `scanner-core`, `attack-engine`, `analysis-LLM`, `ui-gateway`.
2. **Backend & API (день 1–2)**
   - Разработать основной backend-сервис (FastAPI/NestJS/Go) с REST/gRPC API.
   - Реализовать оркестратор задач сканирования/атак, авторизацию, хранение конфигураций.
   - Подготовить SDK/клиент для фронтенда и внутренних модулей.
3. **Интеграция базовых сканеров (день 1–2)**
   - Обёртки для `nmap`, `Shodan`, `VirusTotal`, пассивный DNS/WHOIS.
   - Хранилище результатов (PostgreSQL/ClickHouse) + очередь задач (RQ/Celery).
4. **LLM-сканер и аналитика (день 2)**
   - Парсер веб-страниц (Playwright/Requests+BS4) → нормализация → prompt-цепочка.
   - Генерация отчётов, извлечение метаданных, маппинг на категории БДУ ФСТЭК.
5. **Модуль атак (день 2–3)**
   - Интеграция Metasploit RPC или аналогичного набора эксплойтов.
   - Брутфорс (Hydra/pwntools), SQLi-раннер (sqlmap API), обработка устаревших версий.
6. **Визуализация и интерфейс (день 3)**
   - Web UI (Next.js/React + Tailwind) или Telegram-бот как fallback.
   - Дешборды: статус сканеров, граф инфраструктуры, таблица уязвимостей с фильтрами.
7. **Сборка демо и презентация (конец дня 3)**
   - Скрипты воспроизведения, демонстрационный стенд, скринкасты/слайды.

## Распределение задач (4 разработчика)

1. **Dev A — Core & интеграции**
   - Архитектура модулей разведки и атак, контрактов между сервисами.
   - Настройка БД/очередей, схема данных и единый формат событий.
   - Обёртки для `nmap`, `Shodan`, `VirusTotal`, пассивный сбор данных.
   - API агрегации результатов и нормализация схемы.
2. **Dev B — Attack Engine**
   - Интеграция Metasploit RPC, пайплайн запуска эксплойтов.
   - Реализация брутфорса (Hydra/кастом), SQLi-модуль, проверки устаревших версий.
   - Логика выбора модели тестирования (Black/Grey/White box) и контроль прогресса атак.
3. **Dev C — LLM & аналитика**
   - LLM-сканер (парсер страниц, prompt design, отчёты).
   - Пост-обработка результатов, категоризация по БДУ ФСТЭК, рекомендации.
   - Подсказки внутри UI/бота, формирование текстов презентаций и summary.
4. **Dev D — Backend приложений**
   - Реализация основного API (REST/gRPC), авторизации и управления сессиями.
   - Оркестратор задач сканирования/атак, профили конфигураций Black/Grey/White box.
   - Интеграция LLM-сервиса, Metasploit RPC и внешних сканеров в единый пайплайн.
   - Обеспечение журналирования, очередей заданий и мониторинга.
5. **Dev N — Frontend & визуализация**
   - Web UI/Telegram-бот, авторизация и routing.
   - Компоненты визуализаций (граф сервисов, тепловая карта рисков, таймлайн задач).
   - Интеграция с backend API, экспорт отчётов (PDF/Markdown).

## Бэклог задач (приоритет MVP)

1. Настроить инфраструктуру проекта (репозиторий, CI, Docker-компоуз).
2. Реализовать backend API и единый формат данных разведки/атак.
3. Подключить внешние сканеры и собрать тестовый датасет.
4. Запустить LLM-сканер и добавить оценки БДУ.
5. Собрать атаки (брутфорс, SQLi, эксплойты) под три модели тестирования.
6. Сделать визуализацию результатов и демо-флоу для жюри.
